Preguntas

  1. ¿Qué es el DOM?

    El DOM (Document Object Model) o Modelo de Objetos del Documento es una interfaz de programación para documentos HTML y XML. Representa la estructura de un documento como un árbol de objetos, donde cada nodo es una parte del documento (como un elemento, texto o comentario).

    El DOM proporciona una representación estructurada del documento y define la manera en que la estructura puede ser accedida y manipulada a través de JavaScript, permitiendo así la creación de páginas web dinámicas e interactivas.

    Ejemplo práctico:

    Considere el siguiente fragmento HTML:

    <div id="contenedor">
    <h1>Título</h1>
    <p>Un párrafo de texto</p>
</div>

    En JavaScript, podemos acceder y manipular estos elementos:

    // Acceder a un elemento por su ID
const contenedor = document.getElementById('contenedor');

// Modificar el contenido de un elemento
const titulo = contenedor.querySelector('h1');
titulo.textContent = 'Nuevo título';

// Crear y añadir un nuevo elemento
const nuevoParrafo = document.createElement('p');
nuevoParrafo.textContent = 'Este es un nuevo párrafo';
contenedor.appendChild(nuevoParrafo);

    Referencias:

    • MDN Web Docs. (2023). Introduction to the DOM.
    • W3C. (2004). Document Object Model (DOM) Level 3 Core Specification.
  2. ¿Qué es un nodo?

    Un nodo es la unidad básica dentro de la estructura de árbol del DOM. Cada parte del documento HTML es representada como un nodo: elementos, atributos, texto, comentarios e incluso el documento mismo.

    Los nodos están organizados jerárquicamente, formando lo que se conoce como el "árbol del documento". Cada nodo puede tener un nodo padre y múltiples nodos hijos, estableciendo así relaciones entre los diferentes componentes del documento.

    Tipos principales de nodos:

    • Nodo Documento (Document Node): La raíz del árbol DOM, representando el documento HTML completo.
    • Nodo Elemento (Element Node): Representa etiquetas HTML como <div>, <p>, <a>, etc.
    • Nodo Texto (Text Node): Contiene el texto dentro de un elemento.
    • Nodo Atributo (Attribute Node): Representa los atributos de un elemento HTML.
    • Nodo Comentario (Comment Node): Contiene los comentarios en el HTML.

    Ejemplo práctico:

    Con el siguiente HTML:

    <div id="contenedor" class="principal">
    <!-- Este es un comentario -->
    <p>Texto de ejemplo</p>
</div>

    En este ejemplo tenemos:

    • Un nodo elemento: <div>
    • Dos nodos atributo: id="contenedor" y class="principal"
    • Un nodo comentario: <!-- Este es un comentario -->
    • Un nodo elemento hijo: <p>
    • Un nodo texto: "Texto de ejemplo"

    En JavaScript, podemos navegar entre estos nodos:

    // Acceder al elemento div
const div = document.getElementById('contenedor');

// Acceder a su nodo hijo (el comentario y el párrafo)
const hijos = div.childNodes;

// Acceder al texto dentro del párrafo
const textoParrafo = div.querySelector('p').firstChild.nodeValue;

    Referencias:

    • MDN Web Docs. (2023). Node Interface.
    • Duckett, J. (2014). JavaScript and JQuery: Interactive Front-End Web Development. Wiley.
  3. ¿Cómo se accede a un elemento del DOM?

    Acceder a elementos del DOM es fundamental para la manipulación dinámica de páginas web. JavaScript proporciona varios métodos para seleccionar elementos específicos del documento HTML.

    Métodos principales para acceder a elementos del DOM:

    • getElementById(): Selecciona un elemento por su atributo ID único.
    • getElementsByClassName(): Devuelve una colección de elementos con la clase especificada.
    • getElementsByTagName(): Devuelve una colección de elementos con la etiqueta especificada.
    • querySelector(): Selecciona el primer elemento que coincida con el selector CSS especificado.
    • querySelectorAll(): Devuelve una NodeList con todos los elementos que coincidan con el selector CSS.

    Ejemplo práctico:

    Con el siguiente HTML:

    <div id="contenedor">
    <p class="texto">Primer párrafo</p>
    <p class="texto destacado">Segundo párrafo</p>
    <span>Un span</span>
</div>

    Podemos acceder a estos elementos de diferentes maneras:

    // Por ID
const contenedor = document.getElementById('contenedor');

// Por clase (devuelve una HTMLCollection)
const elementosTexto = document.getElementsByClassName('texto');

// Por etiqueta (devuelve una HTMLCollection)
const parrafos = document.getElementsByTagName('p');

// Por selectores CSS (primer elemento que coincida)
const primerParrafo = document.querySelector('.texto');

// Por selectores CSS (todos los elementos que coincidan)
const todosLosParrafos = document.querySelectorAll('.texto');

// Selectores más complejos
const parrafoDestacado = document.querySelector('.texto.destacado');

    Referencias:

    • MDN Web Docs. (2023). Document.getElementById().
    • MDN Web Docs. (2023). Document.querySelector().
  4. ¿Cómo se modifica un elemento del DOM?

    Modificar un elemento del DOM implica cambiar su contenido, atributos, estilos o estructura. Esto permite crear páginas web dinámicas e interactivas.

    Formas comunes de modificar elementos del DOM:

    • Modificar contenido: Cambiar el texto o HTML dentro de un elemento.
    • Modificar atributos: Cambiar o añadir atributos como id, class, src, href, etc.
    • Modificar estilos: Cambiar las propiedades CSS aplicadas a un elemento.
    • Crear, añadir o eliminar elementos: Manipular la estructura del documento.
    • Modificar clases: Añadir, eliminar o alternar clases CSS.

    Ejemplo práctico:

    Con el siguiente HTML:

    <div id="miElemento" class="contenedor">
    <h2>Título original</h2>
    <p>Texto original</p>
</div>

    Podemos modificarlo de varias formas:

    // Obtener el elemento
const elemento = document.getElementById('miElemento');
const titulo = elemento.querySelector('h2');
const parrafo = elemento.querySelector('p');

// Modificar contenido
titulo.textContent = 'Título modificado';
parrafo.innerHTML = 'Nuevo texto con <b>formato</b>';

// Modificar atributos
elemento.setAttribute('data-info', 'Información adicional');

// Modificar estilos
elemento.style.backgroundColor = 'lightblue';

// Crear y añadir un nuevo elemento
const nuevoParrafo = document.createElement('p');
nuevoParrafo.textContent = 'Este es un nuevo párrafo';
elemento.appendChild(nuevoParrafo);

// Eliminar un elemento
parrafo.remove();

// Modificar clases
elemento.classList.add('destacado');
elemento.classList.toggle('visible');

    Referencias:

    • MDN Web Docs. (2023). Element Interface.
    • MDN Web Docs. (2023). Element.classList.
  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    Ambos métodos permiten acceder a elementos del DOM, pero tienen diferencias importantes en sintaxis, resultados y rendimiento.

    Comparación:

    Aspecto getElementBy* querySelector*
    Sintaxis Usa identificadores específicos (id, clase, etiqueta). Usa selectores CSS (más flexibles).
    Tipo de retorno HTMLCollection o un único elemento. NodeList o un único elemento.
    Compatibilidad Compatible con navegadores antiguos. Requiere navegadores modernos.

    Ejemplo práctico:

    // getElementById
const elemento = document.getElementById('miElemento');

// querySelector
const elementoCSS = document.querySelector('#miElemento');

    Referencias:

    • MDN Web Docs. (2023). Document.querySelector().
  6. ¿Qué es un evento?

    Un evento es una acción o suceso que ocurre en el navegador, como un clic, un movimiento del ratón o la carga de una página. Los eventos permiten que las páginas web sean interactivas.

    Tipos comunes de eventos:

    • Eventos de ratón: click, mouseover, mouseout.
    • Eventos de teclado: keydown, keyup.
    • Eventos de formulario: submit, change.

    Ejemplo práctico:

    // Evento de clic
document.getElementById('miBoton').addEventListener('click', function() {
    alert('¡Botón clickeado!');
});

    Referencias:

    • MDN Web Docs. (2023). Introduction to events.
  7. ¿Qué es un listener?

    Un listener es una función que "escucha" un evento y se ejecuta cuando este ocurre. Se asocia a un elemento del DOM y a un tipo de evento específico.

    Ejemplo práctico:

    // Listener para un clic
document.getElementById('miBoton').addEventListener('click', function() {
    console.log('Botón clickeado');
});

    Referencias:

    • MDN Web Docs. (2023). EventTarget.addEventListener().
  8. ¿Qué es un callback?

    Un callback es una función que se pasa como argumento a otra función y se ejecuta después de que esta última complete su tarea. En el contexto de eventos, los callbacks son las funciones que se ejecutan cuando ocurre un evento.

    Ejemplo práctico:

    // Callback en un evento
document.getElementById('miBoton').addEventListener('click', function() {
    console.log('Este es un callback ejecutado tras el clic');
});

    Referencias:

    • Eloquent JavaScript. (2023). Higher-order Functions.
  9. ¿Qué es un objeto de evento?

    Un objeto de evento es un objeto que contiene información sobre un evento que ha ocurrido, como el tipo de evento, el elemento que lo generó y las coordenadas del ratón (si aplica).

    Ejemplo práctico:

    // Uso del objeto de evento
document.addEventListener('click', function(evento) {
    console.log('Tipo de evento:', evento.type);
    console.log('Elemento objetivo:', evento.target);
});

    Referencias:

    • MDN Web Docs. (2023). Event.
